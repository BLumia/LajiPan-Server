但我到底应该写什么？
  • 对于该实习项目，我们不是要做：
  • - 可以使用在生产环境的产品（考虑该问题的根本原因）
  •  
  • 所以，基本功能同等的情况下：
  • + 可扩展性 > 完整的实现
  • + 更多的解决方案 > 更多的重复代码
选择方案？
  • WebDAV (Distributed Authoring and Versioning) - 分布式对用户可见
  • RAW-text protocol - Pros 'n' Cons
  • Protobuf vs JSON - JSON 大获全胜
  • WebAPI - Why not?
最终决定：
  • 服务器: Muduo - 硬点的
  • 客户端: Qt 全家桶 - 更方便移植到各个平台
  • TCP Protocol API* - 保留原有功能
  • WebAPI - 无限可扩展性和无限可能性
  •  
* 没选 JSON 的确是个错误
